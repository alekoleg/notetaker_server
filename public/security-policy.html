Security Policy

Last updated: 2026-05-07

We take the security of our service, our users' data, and our systems seriously. This document describes the security practices we follow and explains how researchers, users, and partners can responsibly report security issues to us.

Scope

This policy applies to the Bible Note mobile application, the public API, backend infrastructure operated by us, and any official websites under our control. Third-party services used by the application are governed by their own security and privacy terms.

Data Protection

  • All network traffic between the application and our servers is encrypted in transit using TLS 1.2 or higher.
  • User credentials are never stored in plain text. Authentication tokens are issued and validated server-side.
  • Sensitive data at rest is stored on managed infrastructure with encryption enabled at the storage layer.
  • We follow the principle of least privilege for access to production systems and user data.
  • We collect only the data required to operate the service and retain it no longer than necessary.

Access Control

  • Access to production systems is limited to authorized personnel and protected by multi-factor authentication.
  • Administrative actions are logged and reviewed.
  • API access is authenticated and rate-limited to mitigate abuse.

Application Security

  • We follow secure development practices including code review, dependency management, and static analysis.
  • Third-party dependencies are kept up to date and monitored for known vulnerabilities.
  • User-generated content is validated and sanitized before processing or storage.
  • Build and release pipelines are protected and produce signed artifacts for distribution.

Infrastructure Security

  • Production services run on hardened, regularly patched systems.
  • Network access to backend services is restricted by firewalls and security groups.
  • Backups are performed regularly and stored securely to support disaster recovery.
  • Logs are retained for monitoring, auditing, and incident investigation.

Incident Response

In the event of a confirmed security incident affecting user data, we will:

  • Investigate the issue and contain the impact as quickly as possible.
  • Remediate the root cause and verify that affected systems are secure.
  • Notify affected users and, where required, the relevant authorities in accordance with applicable law.

Reporting a Vulnerability

If you believe you have found a security vulnerability in our service, we encourage you to report it to us privately so that we can address it before any details are publicly disclosed.

Please send your report to security@biblenote.app and include the following information where possible:

  • A clear description of the issue and its potential impact.
  • Steps to reproduce, including any required accounts, requests, or payloads.
  • The affected version, endpoint, or platform (iOS, Android, API).
  • Your name or handle if you would like to be credited.

Responsible Disclosure Guidelines

We ask security researchers to:

  • Give us a reasonable amount of time to investigate and remediate the issue before any public disclosure.
  • Avoid privacy violations, data destruction, service disruption, and degradation of user experience.
  • Only interact with accounts that you own or for which you have explicit permission from the account owner.
  • Not exploit a vulnerability beyond what is necessary to demonstrate the issue.

We will not pursue legal action against researchers who act in good faith and follow these guidelines.

Out of Scope

  • Reports based solely on automated scanner output without a demonstrated impact.
  • Missing security headers, cookie flags, or best-practice recommendations without a working exploit.
  • Denial-of-service attacks, social engineering, and physical attacks.
  • Vulnerabilities in third-party services not operated by us.

User Responsibilities

To help keep your account and data safe, we recommend that you:

  • Use a strong, unique password and keep your authentication credentials private.
  • Keep your operating system and the application updated to the latest version.
  • Install the application only from official stores (App Store, Google Play).
  • Contact us immediately if you suspect unauthorized access to your account.

Changes to This Policy

We may update this Security Policy from time to time to reflect changes in our practices, technology, or legal requirements. Updates will be published on this page with a new "Last updated" date.

Contact

For security-related questions or reports, please contact us at security@biblenote.app.
